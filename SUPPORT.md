# Support

## Getting Help

OpenTone is an open-source, offline-first desktop music player. Community-based support is available through the following channels:

### Documentation

- **[README.md](./README.md)** — Overview, features, and getting started guide
- **[docs/architecture.md](./docs/architecture.md)** — Technical architecture and data flow
- **[docs/release-builds.md](./docs/release-builds.md)** — Platform-specific installer details
- **[docs/legal-positioning.md](./docs/legal-positioning.md)** — Legal considerations and DMCA stance
- **[CHANGELOG.md](./CHANGELOG.md)** — Version history and release notes
- **[ROADMAP.md](./ROADMAP.md)** — Planned development

### GitHub

- **Issues** — [Open a bug report or feature request](https://github.com/sparshsam/OpenTone/issues)
- **Discussions** — Use the Discussions tab for questions, ideas, and community conversation
- **Security** — Report vulnerabilities via [GitHub Security Advisories](https://github.com/sparshsam/OpenTone/security/advisories)

### Before Opening an Issue

1. Check the [documentation](./docs) for existing guidance.
2. Search [existing issues](https://github.com/sparshsam/OpenTone/issues) for similar reports.
3. Ensure you are running the latest release: [Releases page](https://github.com/sparshsam/OpenTone/releases).

## Community Guidelines

- Be respectful and constructive.
- Search before posting — someone may have already answered your question.
- Provide detailed information when reporting issues (OS, app version, steps to reproduce).
- Remember that OpenTone is maintained by volunteers. Responses may take time.

## Commercial Support

Commercial support is not currently available. OpenTone is a personal project maintained by its creator.
